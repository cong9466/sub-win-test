# Issue Tracker

Windows UISD 问题跟踪 skill，支持问题记录与查询两大核心流程。

## 流程一：记录问题

当用户输入包含 **"记录问题"** 时触发。

### 步骤

1. **引导用户输入问题要素**

   使用 `question` 工具，依次采集以下字段（一次性询问所有字段）：

   | 字段     | 是否必填 | 默认值   | 说明                         |
   |----------|----------|----------|------------------------------|
   | 问题描述 | 必填     | -        | 详细描述遇到的问题           |
   | 问题来源 | 必填     | -        | 问题的来源模块/场景，如"安装"、"升级"、"UI渲染"等 |
   | 进展     | 必填     | 定位中   | 当前进展：定位中 / 已定位 / 修复中 / 已修复 / 已关闭 |
   | 根因     | 选填     | 待分析   | 问题的根本原因               |
   | 责任人   | 选填     | 待指定   | 负责跟进的人员               |

   使用 question 工具询问用户时，将上述字段作为问题一次性提问，示例：

   ```
   请提供以下问题信息：
   - 问题描述（必填）：请详细描述遇到的问题
   - 问题来源（必填）：如 安装、升级、UI渲染、网络、性能 等
   - 进展：定位中 / 已定位 / 修复中 / 已修复 / 已关闭（默认：定位中）
   - 根因（可选）：如已知根因请填写
   - 责任人（可选）：负责跟进的人员
   ```

   如果用户在触发指令时已经附带了部分信息（如"记录问题：xxx功能异常"），将其作为问题描述，仅询问缺失的必填字段。

2. **生成问题标题**

   根据用户提供的问题描述，自动总结成一个简短摘要作为问题标题，**不超过 30 个字**。标题应简洁概括问题核心。

3. **创建问题文件夹与 README.md**

   - 在项目根目录下的 `issues/` 目录中创建文件夹，命名格式：`YYYYMMDD_问题来源_问题标题`
     - 日期使用当天日期
     - 问题来源和问题标题中的空格替换为下划线
     - 文件夹名不含特殊字符（移除 `/\:*?"<>|` 等）
   - 在该文件夹下创建 `README.md`，按以下模板填充内容：

   ```markdown
   # 标题
   {问题标题}

   ## 描述
   {问题描述}

   ## 来源
   {问题来源}

   ## 责任人
   {责任人，未提供则填"待指定"}

   ## 进展
   {进展状态}

   ## 根因
   {根因，未提供则填"待分析"}
   ```

4. **更新 ISSUES.md**

   - 检查项目根目录下是否存在 `ISSUES.md`
   - 如不存在，创建该文件并写入标题头：
     ```markdown
     # Issues List

     | 日期 | 来源 | 摘要 | 进展 | 目录 |
     |------|------|------|------|------|
     ```
   - 将新问题追加到表格末尾，格式：
     ```
     | YYYY-MM-DD | {来源} | {不超过100字的问题摘要} | {进展} | [查看](issues/YYYYMMDD_来源_标题/) |
     ```
   - 问题摘要根据问题描述生成，**不超过 100 字**。

5. **打开问题文件夹**

   使用 `bash` 工具执行 `explorer` 命令打开刚创建的问题文件夹（Windows 环境）：
   ```bash
   explorer "issues\YYYYMMDD_来源_标题"
   ```

6. **反馈用户**

   告知用户问题已记录成功，显示问题标题、文件夹路径。

## 流程二：查询问题

当用户输入包含 **"查询问题"** 时触发。

### 步骤

1. **获取查询关键词**

   从用户输入中提取查询关键词。如果用户仅输入"查询问题"而没有附带搜索条件，使用 question 工具询问用户想搜索的问题关键词。

2. **加载 ISSUES.md**

   使用 Read 工具读取项目根目录下的 `ISSUES.md` 文件。如文件不存在，直接告知用户"暂无历史问题记录"。

3. **搜索匹配**

   在 ISSUES.md 的内容中，对每条记录的摘要、来源进行模糊匹配，查找与用户输入关键词语义相近的历史问题。匹配逻辑：
   - 先按关键词在摘要和来源列中做子串匹配
   - 如子串无匹配，进行语义相关性判断（如用户搜"崩溃"可匹配"闪退""crash"等）

4. **返回结果**

   - **有匹配**：列出所有匹配的问题记录（日期、来源、摘要、进展、目录链接），并使用 bash 执行 `explorer` 打开第一个匹配问题的文件夹。
   - **无匹配**：回答"未找到与 '{关键词}' 相关的历史问题。"

## 注意事项

- 所有文件路径使用项目根目录（工作目录）作为基准
- 文件夹命名中的日期格式严格为 `YYYYMMDD`（如 `20260323`）
- ISSUES.md 中日期格式为 `YYYY-MM-DD`（如 `2026-03-23`）
- 问题标题和文件夹名中移除所有不适合作为文件名的特殊字符
- 使用 `explorer` 命令打开文件夹（仅适用于 Windows）
